## 6.1 챕터 소개

- 기본적인 리팩토링 (Fundamental Refactoring)

- 프로그래밍에서 가장 중요한 것 ?

  - 변수 : 어떤 데이터가 들어있는지

  - 함수 : 어떤 일을 하는지 한눈에 이해할 수 있도록

  - 모듈 : 변수와 함수를 한 곳에 모아두는 모듈은 어떤 책임을 가지고 있는지

- 변수와 함수는 Low-Level

  - 프로그래밍에서 가장 기본적이고, 중요한 내용

- 모듈은 Hight-Level

## 6.2 함수 포인트 정리

```js
let a = 0;
// 변수이름 Variable name
// 식별자 identifier
```

- 코드를 작성하는 건, 사람, 개발자

- 사람이 **이해하기 쉽도록** 작성

- 함수

  - 특정한 일을 수행하는 코드의 집합

  - 프로그래밍을 이루는 가장 기본적인 단위

  - 프로그램을 작은 단위로 나누는 주된 수단

- 함수 없이 그냥 길게 코드를 작성했거나 함수 안에서 길게 코드를 작성한 경우에는 함수로 만들어 나가는 것이 좋다

  1. 코드가 어떤 일을 하는지 파악한다

  2. 독립적인 함수로 추출하기

  3. 목적에 맞는 이름 짓기

- 함수를 사용했을 때의 가장 큰 장점은 재사용 가능하다는 것

- 그리고 유지보수성도 높아진다

- 또한 의미있는 이름을 부여함으로써 어떤 일을 하는지, 가독성을 높일 수 있다

- 그렇기때문에 함수 내부 구현 사항에 대해서는 매번 신경쓸 필요가 없다

## 6.8 이름짓기 포인트 정리

### 변수

- 변수는 무엇을 담고 있는지 잘 나타낼 수 있어야 한다

  - 보통 명사로 이름 짓는다

  - 책의 예재와 달리 요즘에는 a, an 잘 붙이지 않는다

### 함수

- 함수는 무엇을 하는지 잘 나타낼 수 있어야 한다

  - 보통 동사로 이름을 짓는다

  - 클래스 안에서 getter를 가지고 있는 경우, 그리고 속성에 관련된 함수인 경우에는 함수 이름에 get을 생략할 수 있다

    - getTotalPrice -> totalPrice

  - 인자는 어떤 데이터가 필요한지 잘 나타낼 수 있어야 한다

    - 매개 변수 이름도 잘 지어야 한다

## 6.13 캡슐화 포인트 정리

- 클래스 안에 필요한 데이터를 잘 숨겨두는 개념

- 내부 구현 사항을 숨기고, **외부에서 필요한 사항들만 공개**

- 아래와 같은 객체가 있다면 아무나 손쉽게 잘못된 값으로 이 변수를 변경할 수 있다

```js
let owner = {
  name: 'kim',
};
```

- 이것을 방지하기 위해 모듈, 클래스와 같은 형태로 캡슐화 해두는 것이 좋다

- 데이터를 변경하고 사용하는 코드를 감시할 수 있는 유일한 통로로 만들 수 있다
